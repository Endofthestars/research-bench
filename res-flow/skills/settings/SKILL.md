# 设置面板(交互式浏览与修改)

`.claude/research-bench.config.md` 的**交互式**管理入口,体验对标 Claude Code 内置 `/config`
设置面板:不需要记键名,用 `AskUserQuestion` 菜单逐层选择要看/要改的配置项。
与 `config` 的关系:**同一套数据与写入规则,不同交互形态**——`config` 面向"知道要改什么"
的定点命令式调用,本 skill 面向"浏览着改"的菜单式调用。

> **所属模块**:—(管理配置,同 config);**执行方式**:主流程执行(多轮交互)。
> **开场**:读项目 `.claude/research-bench.config.md` 顶部 frontmatter manifest;
> 无 config → 提示先运行 `/rf:init`(交互)或 `/rf:config init <键值对>`(非交互),终止。

## 面板结构(菜单循环)

每轮用 `AskUserQuestion` 呈现,选完执行,再回主菜单,直到用户选「退出」。

**第 1 层:主菜单**(按 manifest 只列已启用模块的段,未启用的不出现):

| 选项 | 内容 |
|---|---|
| 总览 | 等价 `config` 查看模式:manifest + 各段关键值 + 占位残留统计(只读) |
| manifest(modules / exec-profile) | 只展示当前值;要改 → 说明后**转介 `init`**(牵连装配,面板不代改) |
| §N <段名>(每个已启用段一项) | 进入第 2 层 |
| 保护机制 env | 进入 env 子面板(见下) |
| 退出 | 结束面板 |

**第 2 层:一轮多问改值(默认模式)**:进入某段后,不再"选一个键 → 改一个值 → 回列表"
逐个来,而是利用 `AskUserQuestion` 单次调用最多 4 个问题的能力,一屏同时呈现多个键:

1. **分组**:把该段可定点修改的键按 config 中的出现顺序切成每组**最多 4 个**;
   超过 4 个键则分多轮,每轮开头注明进度(如「§5 第 2/3 轮」)。
2. **每个键一个问题**,`header` 用键名,问题正文附该键的一句说明(取自 config 内该行的
   注释/上下文)与当前值(占位未填的标注 `<未填>`)。选项固定结构:
   - 第 1 项:**「保持当前值:<当前值>」**——语义为**跳过该键**,不产生改动、不需要任何确认;
   - 其余项:有限取值键(布尔/枚举)列合法值;自由文本键列本地扫描到的候选值;
   - 手动输入依赖 `AskUserQuestion` 自动附带的「Other」,不必自己再加一项。
3. **收集与逐值确认(第 3 层)**:一轮回答收齐后,过滤掉「保持当前值」,对**每一个实际有
   变更的键**依次执行现有写入规则:逐个展示 **旧值 → 新值**,单独确认、单独写入——
   选中新值只是"提出改动",**不视为写入授权**,不得把整轮回答静默批量落盘。
   全部确认完(或用户放弃部分)后进入下一轮分组或回主菜单。
   写入规则、机器可读字段同步(如 `source-dir` 双处一致)、职责边界与 `config` 的模式二
   **完全一致**,不另立标准。

若某键取值复杂、候选无法压进 3 个选项,可在该问题里只列「保持当前值」+「修改(进入单键
详改)」,选后者则退化为单键问答(说明 + 输入 + 旧→新确认),不影响同轮其他键。

**env 子面板**:列 `RW_EXEC_PATTERN` / `RW_TRAIN_PATTERNS` 当前值(读项目
`.claude/settings.json` 的 `"env"` 与 config §7.1)。「一轮多问」**仅用于选值**:可一轮
同时问两个键(同样含「保持当前值」项),但每个有变更的键**必须逐个**走 `config` 模式二
特例的全部规则(展示 JSON 片段确认、回填 §7.1、guard hook 二次确认属设计行为、放宽拦截面
显式风险提示),env 改动**绝不因一轮多问而合并或简化确认**。

## 直达参数(可选)

`settings <段号或关键词>`(如 `settings 5`、`settings tracking`)→ 跳过主菜单直接进对应
段的第 2 层;`settings env` → 直达 env 子面板。无法匹配 → 回主菜单。

## 边界与转介(与 config 同表)

| 想做什么 | 走哪 |
|---|---|
| 增删模块 / 换 exec-profile | `init`(面板内选中时说明并转介,不代改) |
| 改脚手架/脚本/skill/hook/checklist | `update-workflow`(steward 两段式) |
| 知道键名的定点改值 / 非交互初始化 | `config`(更快);本面板也能改,规则同源 |

## 规则

- 只写 `.claude/research-bench.config.md` 与(env 子面板,经确认)`.claude/settings.json`
  的 `"env"`;**其余内容均不修改**。
- 每次写入前展示旧值→新值并等待确认;浏览/总览绝对只读。
- 一次会话可连续改多个值,「一轮多问」也只是**一次收集多个改动意向**;写入侧仍**每个值
  单独确认单独写入**,不做批量静默写;「保持当前值」= 跳过,无需确认。
- 不删模块段、不动用户未选中的其他值。
