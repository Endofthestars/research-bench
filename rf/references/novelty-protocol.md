# 查新协议(check-novelty / reviewer 共用)

> 查新要在投入实验前回答一个问题:这个方向是否已被做过、最接近的前期工作差在哪。
> 本协议规定:四阶段流程、反幻觉规则、判定标准。check-novelty skill 与 reviewer agent
> 各自引用本文件,不重复内联。证据读取仍遵 `evidence-protocol.md`(数据源优先级含文献维度);
> 检索源、近月窗口与分级门槛等项目参数在 config §12.4,修改它们属于削弱检查标准,
> 须显式说明并经用户确认。

## 1. 四阶段流程

### 阶段一:核心主张提取
- 从候选方向(种子想法 / 选中的假设卡片 / 方向档案 direction.md)提取 **3–5 个核心主张**(claim):
  每条一句话,写明「什么方法 + 作用于什么问题 + 预期什么效果」。
- 主张超过 5 条说明方向未收敛,先回 `refine-direction`;不足 3 条说明表述过粗,继续拆分。
- 区分**方法主张**与**实验设置主张**(双层新颖性,见 §3),两层分别查新。

### 阶段二:多源检索
- 检索源:config §12.4 的检索源清单(默认 arXiv / Semantic Scholar / CrossRef + WebSearch)。
- 每个主张至少 **3 种查询表述**:① 直译关键词;② 同义改写(领域别名、方法族名);
  ③ 上位概念 + 应用场景组合。只用一种表述漏检率高,不得省略。
- **强制检索近 6 个月 arXiv**(窗口按 config §12.4,默认 6 个月):撞车风险最高的是未刊预印本。
- 检出条目按 §2 落库或核验;未通过验证的条目不得进入阶段四的对照表。

### 阶段三:跨模型交叉验证
- 主流程将主张清单 + 阶段二检出的最接近工作交给**外部模型家族**取独立裁决:经 config §12.2 的
  Codex 通道按固定问题清单执行——新颖吗 / 最接近的工作是什么 / 差异何在 / 评分 X/10。
- 目的是避开自我偏好:检索与判定出自同一模型时,容易系统性漏掉否定自己的证据。
- 未配置 Codex 通道时降级:由独立 reviewer agent 冷上下文执行同一问题清单,裁决注明「单签(降级)」。
- 交叉验证裁决与主流程检索结论**双签**记入 novelty.md 的裁决原文区;
  两方矛盾时如实并列,不得只保留有利一方。

### 阶段四:分级判定
- **每主张分级**(高 / 中 / 低):
  - **高**:3 种表述 + 近月窗口检索均无实质重合命中;
  - **中**:存在相近工作,但方法或实验设置至少一层有实质差异(差异点写进对照表);
  - **低**:检出实质重合,或差异仅为 §3 所列「不构成创新」的形态。
- **最接近前期工作对照表**(写入 novelty.md;每行必须带 Zotero key,见 §2):
  `| 前期工作(Zotero key)| 重合点 | 差异点 | 威胁等级(高/中/低)|`
- **0–10 总分 + 建议**:继续(达到 config §12.4 门槛)/ 谨慎(边缘分,须列出待补的差异化论证)/
  放弃(撞车;建议把「已被某论文做过」条目回写教训库,附撞车论文的 Zotero key)。
- 判定与总分写入 novelty.md,并向 gates.jsonl 追加一行 `{"gate":"novelty",…}`
  (schema 见方向模板 direction.md 末尾注释;写入由主流程经 update-workflow 或用户确认执行)。

## 2. 反幻觉规则(引用可验证性)

- **「已验证」的定义 = 按 DOI / arXiv id 成功导入 Zotero**(导入即完成解析验证)。
  对照表每行必须带 Zotero key,幻觉引用物理上过不去。集合约定见 config §12.1
  (`directions/<slug>`,每方向一集合)。
- **无 Zotero 时降级为三层核验**:arXiv API → CrossRef → Semantic Scholar,按 id 任一层解析成功
  即算核验通过,对照表的 Zotero key 列改填「核验来源 + id」;三层都验不过的引用**必须标
  `[UNVERIFIED]`**,不得作为分级判定或否决的依据。
- **禁止凭记忆编造 DOI / arXiv id / 作者-年份**:引用只能来自本次检索结果或库内已验证条目。
  记忆/印象禁止作为证据(与 evidence-protocol.md §2 同族条款)。
- 检索到但未落库(或未过三层核验)的条目视为**未验证**:只能作为线索继续检索,不得进对照表。

## 3. 判定标准

- **「应用 X 于 Y」不构成创新**,除非产生意外洞见(如揭示 X 在 Y 上失效的机制、
  或 Y 的结构迫使 X 产生非平凡变体)。单纯换数据集 / 换领域跑通,判「低」。
- **双层新颖性**:方法新颖性(干预本身是否已有)与实验设置新颖性(评估协议 / 对照设计 /
  指标视角是否已有)分开判。仅实验设置新颖时须显式标注——论文定位随之不同。
- 增量式改进(调参、换 backbone、加常见 trick)不因「组合未见过」而判「高」:
  组合的新颖性要求组合本身产生单项没有的性质。
- **检索不到 ≠ 新颖**:冷门表述可能漏检。判「高」前必须穷尽阶段二的 3 种表述;
  仍无命中时在 novelty.md 注明检索式与检索时间,供日后复查。
