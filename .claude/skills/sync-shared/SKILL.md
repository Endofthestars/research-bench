# 共享内容同步

`shared/` 是 rf、ef 共有文件的唯一事实来源(`{{P}}` = 插件前缀),插件内副本是生成物。

## 步骤

1. 运行 `scripts/sync-shared.sh --check`。
2. **无漂移** → 报告"一致",结束(参数为 `check` 时到此为止)。
3. **有漂移** → 对每个漂移文件判断方向:
   - `git diff` 查看是谁改了:若改动发生在**插件副本**(有人忘了规矩直接改了生成物),
     把改动**回收进 `shared/` 对应文件**(前缀位置改写为 `{{P}}`;若 rf、ef 两侧副本被改得互相矛盾,
     停下来把两版差异展示给用户,让用户裁决);
   - 若改动发生在 `shared/` 本身 → 正常情况,直接进入下一步渲染。
4. 运行 `scripts/sync-shared.sh` 渲染,再 `--check` 确认零漂移。
5. 输出:漂移文件清单、回收/覆盖了什么、最终校验结果。

## 规则

- 任何会**丢弃**插件副本中人工改动的覆盖操作,先展示 diff 并经用户确认。
- 新文件想进入共享集:放入 `shared/` 对应相对路径即可,渲染会自动落到两插件。
