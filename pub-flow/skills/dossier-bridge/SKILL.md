# dossier-bridge(rf 方向档案 → ARS 写作素材包)

把 **一个** rf 方向 dossier 的结构化证据(claims 表、新颖性对照表、审查薄弱论点)翻译成
ARS(academic-paper / academic-pipeline)入口可直接消费的**英文 materials brief**。
本 skill 是 research-bench 的本土原创(不属上游 ARS),只做「读档案 → 整理素材」,
不写论文、不查新、不跑实验。

> **接口契约**:只读 rf 侧文件(方向 dossier + config),只写稿件工作区
> `docs/paper/<paper-slug>/`;dossier 的字段名(slug / status / claims 列结构)是 rf 的
> 既有契约,本 skill 是其消费方,不得回写、不得改名。

## 使用时机
- 方向 dossier 已 `status: active`,claims 表有 `done` 行,想开始起草论文
- 已装 rf 并走完三关(novelty / pilot / review),要把证据链接进 ARS 写作流水线
- 未装 rf 或未启用 directions 也可用:走下文降级路径,手工指认素材

## 流程

### 1. 读配置,定位 dossier
读项目 `.claude/research-bench.config.md`:
- **§10 方向/路线注册表** → 方向文件目录(如 `docs/directions/<slug>/`);
- **§12.1 Zotero 文献库** → 集合约定(如 `directions/<slug>`,一方向一集合)与未配置时的降级条款。

**降级**:config 缺失、或 §10 / §12.1 缺席 → 不中止,请用户手工指认:
① 方向目录(或任何包含主张/证据/文献线索的素材文件);② 文献线索来源
(Zotero 集合名,或「核验来源 + id」清单)。降级路径下后续步骤照走,来源如实标注。

用户未指定方向时,列出方向目录下的候选 slug(优先 frontmatter `status: active` 者)请用户选择。
**一次桥接一个方向**;一篇论文合并多方向时逐个执行本流程,素材追加进同一个 materials.md
(每方向一节,注明来源 slug)。

### 2. 读方向 dossier(只读)
在 `docs/directions/<slug>/` 下:
- **direction.md**:
  - frontmatter `status` 应为 `active`;不是则警告(说明该方向未过全部关卡)并请用户确认是否继续;
  - **claims 表**:只取 `status=done` **且证据指针非空**的行(planned/running 行、无证据指针的行
    一律不进素材包——稿件主张不得超前于证据);
  - 「核心诊断」(现状瓶颈 / 改动假设 / 可证伪预测)与「目标」→ 研究背景素材;
  - 「文献」节的 Zotero 集合指针与关键引用 key 清单 → 文献素材。
- **novelty.md**:「最接近前期工作对照表」(每行含 Zotero key、重合点、差异点、威胁等级)
  → 相关工作素材;总分与建议一并记录。
- **review.md**:「薄弱论点」表(主张原文 / 为什么站不住 / 审稿人攻击面)
  → 写作时要主动回应的审稿关切;「实验设计建议」中已完成的补强也值得带上。

某文件缺失(如未走完三关)→ 对应素材节标注 `(not available: <原因>)`,不编造内容。

### 3. 生成英文素材包
问用户 paper-slug(默认 = 方向 slug),写入 `docs/paper/<paper-slug>/materials.md`。
**素材包全文英文**(ARS skill 主体是英文,且素材会被直接引用进英文稿件),结构如下:

```markdown
# Materials Brief: <paper working title>
> Generated by /pf:dossier-bridge from rf dossier `docs/directions/<slug>/` on <ISO date>.
> Read-only source; do not edit the dossier from the paper workspace.

## Research Context
<从 direction.md 目标 + 核心诊断翻译:现状瓶颈、改动假设、可证伪预测>

## Contributions (from claims, status=done only)
| # | Claim | Supporting experiment (config) | Evidence pointer |
|---|---|---|---|
<claims 表逐行翻译;证据指针原样保留(汇总表行 / tracking run id),不翻译路径与 id>

## Evidence Pointers
<证据指针的展开说明:结果表位置、tracking run、seed 等;
 实验由研究者在 ef 侧完成,ARS 只做 intake 与对账,本节即其原料>

## Related Work (from novelty dossier)
| Prior work (Zotero key) | Overlap | Difference | Threat level |
|---|---|---|---|
<novelty.md 对照表逐行翻译,Zotero key 原样保留;附检索日期与新颖性总分/建议>

## Anticipated Reviewer Concerns (from adversarial review)
| # | Claim under attack | Why it is weak | Reviewer attack surface | Mitigation status |
|---|---|---|---|---|
<review.md 薄弱论点逐行翻译;已按「实验设计建议」补强的注明补强证据>

## Literature
- Zotero collection: `<config §12.1 约定的集合,如 directions/<slug>>`
- Key references: <direction.md 文献节的 key 清单,含角色标注>
```

**引用铁律**(沿 rf novelty-protocol §2):素材包里每条引用必须带 Zotero key;
无 Zotero 时按 §12.1 降级条款改填「核验来源 + id」(如 `arXiv:2605.xxxxx`);
两者都没有的引用**必须标 `[UNVERIFIED]`**,并提醒用户 ARS 写作阶段不得依赖它。
禁止凭记忆补 DOI / key。

写入前把整包给用户过目确认;`docs/paper/<paper-slug>/` 不存在则创建。

### 4. 提示后续入口(ARS 侧)
素材包写好后,向用户说明衔接方式——ARS `academic-paper` 的 intake 面谈自带
「现有素材盘点」步骤(Existing Materials),把 materials.md 指给它即可:

- `/pf:ars-plan` — academic-paper `plan` 模式,苏格拉底式逐章规划(拿不准结构时首选);
- `/pf:ars-outline` — academic-paper `outline-only` 模式,直接产大纲 + 证据映射;
- `/pf:ars-full` — academic-pipeline 十阶段全流程(research → write → review → revise → finalize;
  已有 dossier 证据时 Stage 1 调研可按其中期入口规则跳过或从简);
- 或自然语言触发 academic-paper `full` 模式:
  `Write a paper based on docs/paper/<paper-slug>/materials.md`。

可选进阶:文献量大时,可用 ARS 自带的 Zotero 参考适配器
(`scripts/adapters/zotero.py`,产出 Material Passport `literature_corpus[]`)
把 Zotero 集合正式导入 ARS 文献账本;本 skill 不代跑,只提示。

## 约束
- **只读 dossier**:不写、不改 `docs/directions/` 下任何文件;论文进度回写 dossier
  「结论 / 论文落点」节仍走 rf 的 update-workflow + 用户确认,与本 skill 无关。
- **只写稿件工作区**:产物仅限 `docs/paper/<paper-slug>/` 下。
- **证据不越界**:未 done 或无证据指针的 claim 不进素材包;review.md 的薄弱论点必须如实带入,
  不得因「不利于成稿」而省略。
